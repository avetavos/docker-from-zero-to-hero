## ตัวเลือกการติดตั้ง

Docker มีสองแพ็กเกจหลัก ขึ้นอยู่กับระบบปฏิบัติการของคุณ:

- **Docker Desktop** — แอปพลิเคชัน GUI ที่รวม Docker Engine, Docker Compose, Kubernetes แบบ node เดียว และ dashboard สำหรับ macOS และ Windows
- **Docker Engine** — daemon บรรทัดคำสั่งสำหรับ Linux ไม่มี GUI แต่เบากว่าและเหมาะสำหรับเซิร์ฟเวอร์และ CI

## ติดตั้งบน macOS

ดาวน์โหลด Docker Desktop จาก [docs.docker.com/desktop/install/mac-install](https://docs.docker.com/desktop/install/mac-install/) เลือก chip ของคุณ (Apple Silicon หรือ Intel) จากนั้นลาก Docker.app ไปที่ Applications แล้วเปิดใช้งาน

เมื่อ whale icon ปรากฏในแถบเมนูและแสดงว่า "Docker Desktop is running" daemon ก็พร้อมใช้งานแล้ว

## ติดตั้งบน Windows

ดาวน์โหลด Docker Desktop installer จาก [docs.docker.com/desktop/install/windows-install](https://docs.docker.com/desktop/install/windows-install/) Docker Desktop บน Windows ใช้ WSL 2 (Windows Subsystem for Linux) เป็น backend ซึ่ง installer จะตั้งค่าให้โดยอัตโนมัติหากยังไม่ได้เปิดใช้งาน

รีสตาร์ทเมื่อได้รับแจ้ง จากนั้น Docker Desktop จะเริ่มทำงานในระบบถาดแจ้งเตือน

## ติดตั้งบน Linux

บน Linux คุณจะติดตั้ง Docker Engine โดยตรง วิธีที่แนะนำคือใช้สคริปต์ความสะดวกของ Docker หรือ repository ของแพ็กเกจ manager ของ distro คุณ

```bash
# ตัวอย่างสำหรับ Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

เพื่อให้รัน Docker โดยไม่ต้องใช้ `sudo` ให้เพิ่ม user ของคุณในกลุ่ม `docker`:

```bash
sudo usermod -aG docker $USER
newgrp docker
```

## ยืนยันการติดตั้ง

รันสองคำสั่งนี้เพื่อยืนยันว่า Docker client และ daemon ทั้งสองทำงานอยู่:

```bash
docker version
docker run hello-world
```

`docker version` แสดงเวอร์ชันของทั้ง client และ server (daemon) หาก server section หายไปแสดงว่า daemon ไม่ได้ทำงานอยู่

`docker run hello-world` ดึง image ขนาดเล็กจาก Docker Hub รันในคอนเทนเนอร์ พิมพ์ข้อความยืนยัน แล้วออก เป็นการยืนยัน end-to-end ตั้งแต่ registry ไปจนถึง runtime

```text
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

ลองรันคำสั่งเหล่านี้ใน playground ด้านล่าง:
